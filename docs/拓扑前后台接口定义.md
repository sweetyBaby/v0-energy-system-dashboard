# 拓扑图 · 前后台接口定义

> 适用：总览页拓扑图（`ProjectTopologyPanel`）。当前为 **mock 阶段**，后端就绪后按本文对接。
> 相关代码：`lib/topo/topo-api.ts`（取数接缝）、`lib/topo/topo-types.ts`（类型契约）、
> `lib/topo/apply-signals.ts`（合并逻辑）、`components/dashboard/topology/*`（渲染）。
> 渲染/规则细节见 `docs/topology-engine-and-rules.md`。

---

## 0. 一句话

拓扑交互拆成 **两条独立数据流**：

| 数据流 | 含义 | 频率 | 接口 |
|---|---|---|---|
| **布局** `TopologyDoc` | 图的结构：节点/连线/样式/规则定义 | 低频（进页面拉一次） | `GET /topology/layout` |
| **实时信号** `TopoSignals` | 图上的活数据：扁平 `{信号名:值}` | 高频（轮询，默认 2s） | `GET /topology/signals` |

前端 `applySignals(布局, 信号)` 合并后渲染。**这条分界线不要打破**：布局接口不返回实时值，信号接口不返回结构。

## 2. 接口 1：拉取布局

```
GET /topology/layout?projectId={id}
Authorization: <token>          # 走项目统一 apiClient（/api/proxy 代理 + 鉴权）
```

返回 `ApiResponse<TopologyDoc>`（统一包壳 `{ code, message, data }`，见 `lib/api-client.ts`）。
`data` 即运营端编辑器 `buildJSON()` 导出的画布 JSON（`schemaVersion: "2.0"`），**后端原样透传即可**：

```jsonc
{
  "code": 0,
  "message": "ok",
  "data": {
    "schemaVersion": "2.0",
    "meta": {
      "lang": "zh",
      "libraryRef": { "name": "...", "version": "..." },
      "canvas": { "bgColor": "#0a1f40", "zoom": 1, "panX": 0, "panY": 0,
                  "grid": { "show": true, "stepPx": 20 }, "showAnchors": false }
    },
    "edgeStyles": {
      "<edgeType>": { "labelZh": "", "labelEn": "", "color": "#5fd0ff",
                      "width": 2.5, "dash": [], "anim": "flow", "speed": 1 }
    },
    "nodes": [ /* TopoNode[]，见 §4 */ ],
    "edges": [ /* TopoEdge[]，见 §4 */ ],
    "signals": [ /* 全局信号定义，可选 */ ],
    "sampleSignals": { /* 未接实时数据时的默认值，可选 */ }
  }
}
```

要点：
- 节点字段值 `nodes[].data[].value` 用占位 `"--"`，**实时值不放这里**——由信号接口提供。
- `visibleWhen` / `edges[].showWhen` / `dirRules` 是**规则**，随布局下发一次，求值在前端每帧做。
- 节点点击跳转 `nav` **不由后端提供**：前端 `lib/topo/project-topology-config.ts` 按 `type/id` 解析。

## 3. 接口 2：实时信号

```
GET /topology/signals?projectId={id}
```

返回 `ApiResponse<TopoSignals>`，`data` 是**扁平字典** `Record<string, number|string|boolean|null>`。
**同一份信号同时干三件事**：① 回写字段卡片的值 ② 覆盖节点状态文字 ③ 驱动规则（显隐 / 流向）。前端不做任何私有推导。

```jsonc
{
  "code": 0, "message": "ok",
  "data": {
    // ① 节点字段：键 = `节点id.字段名`（字段名 = 布局里 data[].key.zh，含括号单位，原样）
    "pcs_1.P(kW)": -77.9,
    "pcs_1.I(A)": -101.3,
    "pcs_1.U(V)": 690,
    "bms_1.SOC(%)": 65,
    "bms_1.温度(℃)": 28,
    "grid_1.今日用电(kWh)": 2301,

    // ② 节点状态：键 = `节点id.status`，覆盖状态文字（zh/en 同值）
    "pcs_1.status": "运行",
    "bms_1.status": "充电",

    // ③ 在线标志：键 = `节点id.online`，仅参与规则，不作为字段显示
    "gen_1.online": true,

    // ④ 全局信号：键名"最后一个点之前"不是已知节点id → 全局，仅参与规则
    "workMode": "charge"
  }
}
```

### 键名约定（唯一关键契约）

解析规则（`apply-signals.ts::parseSignal`，与运营端 `topo.html` 一致）：取键名里**最后一个 `.`** 拆分，
前缀若是布局中存在的 `节点id` 即为节点信号，否则整名当作全局信号。

| 键名形态 | 含义 | 作用 |
|---|---|---|
| `节点id.字段名` | 节点数据字段（中文 `key.zh`，含单位） | 回写字段卡片值 **+** 参与规则 |
| `节点id.status` | 节点状态 | 覆盖状态文字（zh/en 同值）**+** 参与规则 |
| `节点id.online` | 在线布尔 | **仅**规则，无字段显示 |
| 无匹配节点前缀的名字 | 全局信号 | **仅**规则 |

### 值语义与对齐细节（后端必读）

1. **`节点id` 必须与布局 JSON 的 `nodes[].id` 逐字一致**；前缀对不上会被当成全局信号、静默丢弃（不报错、不显示）。
2. **字段名用中文 `key.zh`（含括号单位）**，如 `P(kW)`、`温度(℃)`、`SOC(%)`——不是英文、不随界面语言变；后端字段 key 须与该节点 `data[].key.zh` 完全相同，否则不回写。
3. **值类型** = 数值 / 字符串 / 布尔；`null` 或空串 `""` 表示"无值"，卡片显示为空（**不是** 0）。
4. **规则引用**：布局里的 `visibleWhen / showWhen / dirRules` 用 `var: "信号名"` 引用这些键。后端按约定塞入信号，显隐 / 流向即自动生效。示例（见 `topology-sample.json` + `mock-signals.ts`）：
   - `solar_1.visibleWhen: P(kW)>0` → 夜间光伏（含连线）整体隐藏；
   - `pcs_1→bms_1.dirRules: I(A)` 正充负放 → 流向随符号翻转；
   - `gen_1→pcs_1.showWhen: gen_1.online` → 发电机离线时联络线消失。
5. **可运行样例**：`lib/topo/mock-signals.ts` 即本契约的样例（驱动上述三条规则）；后端按同结构返回，前端把 `topo-api.ts` 的 `TOPOLOGY_USE_MOCK` 置 `false` 即切真实接口、零改动。

### 增量 vs 全量

前端在 `TopoCanvas` 内对信号**累积合并**（`signalsRef`），所以接口**全量或只发变化的信号都可以**。
轮询周期由 `intervalMs` 控制（默认 2000ms）。

### 升级：WebSocket（可选，后续）

高频/省流时可改为推送，前端只需把接缝层 `makeTopologySignals` 换成订阅、将 patch 合并进信号即可，
`applySignals` 与渲染不变：

```
WS /topology/stream?projectId={id}
← { "type": "signals", "patch": { "pcs_1.I(A)": 12.3 } }
```

## 4. 类型契约（节选，完整见 `lib/topo/topo-types.ts`）

```ts
type Bilingual = { zh: string; en: string }
type Point = { x: number; y: number }

type TopoNode = {
  id: string; type: string; label: Bilingual; position: Point
  sizeWorld: number; scale: number; rotation: number
  status: Bilingual
  display: { showLabel: boolean; showFields: boolean }
  data: { key: Bilingual; value: string | number | boolean; hidden: boolean; offset: Point }[]
  visibleWhen?: TopoCond        // 规则：条件不满足 → 节点及其连线隐藏
}

type TopoEdge = {
  from: string; to: string; edgeType: string
  route: "smart" | "arc" | "manual" | "line" | "straight"
  dir: "forward" | "reverse" | "both" | "none"
  width: number; label: string; showLabel: boolean
  waypoints: Point[]
  showWhen?: TopoCond           // 规则：不满足 → 不画该连线
  dirRules?: { when?: TopoCond; dir: EdgeDir }[]   // 规则：顺序匹配，dir 兜底
}

// 条件树：叶子 {var,op,val|ref}；组合 {all}/{any}/{not}；缺省=恒为真
// op: == != > >= < <= in between truthy falsy exists
type TopoCond = { all?: TopoCond[]; any?: TopoCond[]; not?: TopoCond
                  var?: string; op?: string; val?: unknown; ref?: string }

type TopoSignals = Record<string, number | string | boolean | null>
```

## 5. 错误 / 鉴权

- 走项目统一 `apiClient`：自动带鉴权头、经 `/api/proxy` 代理、401 触发重新登录。
- 布局拉取失败 → 画布显示「拓扑数据加载失败」。
- 信号拉取失败 → 保留上一帧，下个周期自动重试（不闪断）。
